# 1. Purpose of File Handling

The primary goal of this topic is to teach you how to manage data outside of a program's temporary memory. While previous topics focused on data stored in variables and arrays (which are lost when the program ends), **File Handling** allows for **data persistence**—saving information to a file so it can be retrieved and used later.

# 2. Core Sub-Topics

Based on the course syllabus, the module covers three main areas:

- **Files:** Understanding the structure and management of files within the operating system.
- **Reading:** Learning techniques to extract data from existing files into your program.
- **Writing:** Developing the ability to save program output or user data into external files.

# 3. Implementation and Tools

File handling in C is primarily managed through the **standard input-output library (****stdio.h****)**. The course introduces specific functions for these tasks, such as:

- **fprintf()**: A standard library function used for sending formatted output to a file.
- Other functions mentioned in the context of the `stdio.h` header include **getchar()** and **scanf()**, though the latter is typically for console input.

By the end of this module, you are expected to achieve **Intended Learning Outcome 08 (LO8)**: **Understand and apply simple data structures and file handling**. This ensures you can implement and debug applications that solve **real-life problems** where data needs to be stored and processed across multiple sessions.
