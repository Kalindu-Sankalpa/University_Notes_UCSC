> [!Objective]
> This topic covers the **three major components of system development**:
> 1. **Methodology:** A formal and precise process defining activities, methods, best practices, deliverables, and automated tools used to develop or improve software.
> 2. **Modeling Methods or Techniques:** Specific visual representations such as **Class Diagrams** and **Use Case Diagrams**.
> 3. **Tools:** Software used in the development process, such as **Visual Paradigm**, **Rational Rhapsody**, and **Software Ideas Modeler**.
# Software Process

A set of related activities leading to the production of software, witch can be ether
- building from scratch
- modifying existing software or
- integrating off-the-shelf components

# The Four Essential Phases

Regardless of the specific process used, all must have:
- Software Specification
- Software Design and Implementation
- Software Validation
- Software Evolution

# Methodology vs. Process Analogy

In the context of building a house, the **"Process"** consist of the stages a contractor takes plan to completion, while the **"Methodology"** refers to the specific skills and capabilities of the individuals (subcontracts/tradespeople) preforming the work.

# Software Process Models

-  **Waterfall Model:** A linear, sequential model where one phase must be completed before the next begins.
- **Iterative Development:** Systems are developed as a series of versions; a rough version is built and validated, with more quality and functionality added in each subsequent iteration.
- **Incremental Development:** The system is broken into increments; user requirements are prioritized, and high-priority features are delivered in early increments.
- **Reuse-Oriented Software Engineering:** Focuses on integrating a significant number of **reusable components** rather than developing from scratch.
- **Combined Approaches:** In large-scale systems, different subsystems may use different models or combine the best features of several models.

# Summary
## System Development Components :

- ***Methodology:*** Framework/steps (e.g., SSADM).
- ***Techniques:*** Visual models (Class/Use Case Diagrams).
- ***Tools:*** Automated software (Visual Paradigm).

## Mandatory Software Phases :

1. Specification (Requirements).
2. Design & Implementation (Building).
3. Validation (Testing).
4. Evolution (Maintenance).

## Process Model Comparison

- **Waterfall:** Linear/Sequential.
- **Iterative:** Version-by-version; adds quality over time.
- **Incremental:** Part-by-part; delivers high-priority functions first.
- **Reuse-Oriented:** Based on existing components.

> [!NOTE]
> To understand the difference between **Incremental** and **Iterative** development, think of painting a portrait. **Incremental** development is like painting the head completely, then the body, then the background; **Iterative** development is like sketching the whole scene roughly and then adding detail and color to the entire canvas through several passes
