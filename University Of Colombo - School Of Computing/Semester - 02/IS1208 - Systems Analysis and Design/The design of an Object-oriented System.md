>[!tip] Introduction
>Object-oriented (OO) design shifts the focus from "what" a system does to "how" it will technically work. It translates the functional requirements discovered during analysis into a detailed technical blueprint.

---
# The Four Main Design Sections

1. **Architecture Design:** Defines the technical infrastructure, including hardware, software, and network components.
2. **Software Design:** Specifies the programs and individual system modules that need to be constructed.
3. **Database Design:** Develops the technical models for files and database tables required to store the system's data.
4. **Interface Design:** Focuses on the user experience by designing the screens, forms, and reports users will interact with.

# Core Object Oriented (OO) Concepts in Design

• **Encapsulation:** The "packaging" of data (attributes) and behavior (methods) into a single unit. It hides internal logic from other objects to protect the data and simplify changes.

• **Inheritance:** Allows a "child" class (sub-type) to reuse the attributes and methods of a "parent" class (super-type). For example, both _Student_ and _Teacher_ classes can inherit general features from a _Person_ class.

• **Polymorphism:** The ability for different objects to respond to the same message in unique ways. It allows one operation name, like "Open()", to perform different actions depending on whether it is opening a door or a bank account.

• **Message Passing:** The process where objects communicate by invoking each other's methods to request information or actions.

# Short Note for Memorization

- **Goal:** Design is the **"How"** (Technical Solution).
- **Components (ASDI):**
	- **A**rchitecture (Hardware/Networks).
	- **S**oftware (Programs/Modules).
	- **D**atabase (Data Stores).
	- **I**nterface (Screens/Reports).
- **OO Pillars (PIE):**
	- **P**olymorphism (Many forms for one message).
	- **I**nheritance (Reuse from parent to child).
	- **E**ncapsulation (Hiding complexity inside a unit).
